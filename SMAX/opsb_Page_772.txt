Create NFS volumes for Audit
To install the Audit service, you need to export NFS volumes to store the application data.
Create NFS volumes
1
. 
Create an NFS directory for Audit by running the following command:
mkdir -p /var/vols/itom/audit/<Audit Namespace>/vault
mkdir -p /var/vols/itom/audit/<Audit Namespace>/log
where, 
<Audit Namespace>
 is the namespace where you deploy the Audit service.
For example,
mkdir -p /var/vols/itom/audit/auditns/vault
mkdir -p /var/vols/itom/audit/auditns/log
2
. 
Change the permission for the directory that you have created by running the following command:
chmod g+ws /var/vols/itom/audit/<Audit Namespace>/vault
chmod g+ws /var/vols/itom/audit/<Audit Namespace>/log
For example,
chmod g+ws /var/vols/itom/audit/auditns/vault
chmod g+ws /var/vols/itom/audit/auditns/log
3
. 
Change the owner of the directory you have created by running the following command:
chown 
<SYSTEM_USER_ID>
:
<SYSTEM_GROUP_ID>
/var/vols/itom/audit/<Audit Namespace>/vault
chown 
<SYSTEM_USER_ID>
:
<SYSTEM_GROUP_ID>
/var/vols/itom/audit/<Audit Namespace>/log
For example,
chown 1999:1999 /var/vols/itom/audit/auditns/vault
chown 1999:1999 /var/vols/itom/audit/auditns/log
4
. 
To export the NFS directory, add the following values in the 
/etc/exports
 file:
/var/vols/itom/audit/<Audit Namespace> *(rw,sync,anonuid=1999,anongid=1999,root_squash)
For example,
/var/vols/itom/audit/auditns *(rw,sync,anonuid=1999,anongid=1999,root_squash)
5
. 
Run the following command to export the folder:
exportfs -ra
6
. 
Check the exported NFS volumes:
showmount -e
