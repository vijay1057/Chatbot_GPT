In earlier releases, Service Management Automation uses Service Manager as the Service Management module. This release
introduces an "X platform", which can be used as a replacement of Service Manager. During the suite installation, customers
have the option to choose either the Service Manager option or the X platform option, which are referred to as "Service
Management Automation SM (SMA-SM)" and "Service Management Automation X (SMAX)", respectively.

Offers the ability to install and work with SMAX instances deployed on the Amazon Web Services ecosystem, providing

tasks that need to be managed on the go.

Apply simple suite update
You can now check for suite updates on Docker Hub and apply the updates to your suite environment.
Install from a backup
You can now use a backup of an existing environment to set up a replicated environment. During the installation process, the
suite installer allows you to select either a new installation or an installation based on backed up data from an existing
deployment. Note that this feature requires that both CDF and the suite in the two environments be the same version and
patch level.
Enhanced database support
This release adds support of external PostgreSQL for UCMDB.
